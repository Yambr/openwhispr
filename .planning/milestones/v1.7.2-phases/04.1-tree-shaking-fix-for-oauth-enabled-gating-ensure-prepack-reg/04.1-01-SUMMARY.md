# Phase 04.1 Plan 01: Pack/Dist Regenerate Build-Config (CFG-08) Summary

## What Changed

### `scripts/verify-pack-regenerates-build-config.js` (new, 141 lines)

Pure Node script. Strategy:

1. Regenerate cjs with no env → assert `OAUTH_GOOGLE_ENABLED=true` (pre-condition).
2. Read `package.json#scripts.pack`, split on `&&`, run every step EXCEPT `electron-builder`, with `OPENWHISPR_OAUTH_GOOGLE=false` set.
3. Re-read cjs → assert `OAUTH_GOOGLE_ENABLED=false`.
4. Cleanup (always, in `finally`): regenerate with no env so dev tree is clean.
5. Exit 0 = PASS, exit 1 = FAIL with a precise diagnostic line.

This is a self-driving simulation rather than a true `npm run pack` invocation because we don't want CI/dev to actually run electron-builder (slow, irrelevant to the assertion). The test parses the literal script to stay in sync with whatever steps `pack` accumulates over time.

### `package.json`

```diff
-    "pack": "npm run build:renderer && cross-env CSC_IDENTITY_AUTO_DISCOVERY=false electron-builder --dir",
+    "pack": "node scripts/generate-build-config.js && npm run build:renderer && cross-env CSC_IDENTITY_AUTO_DISCOVERY=false electron-builder --dir",
-    "dist": "npm run build:renderer && electron-builder",
+    "dist": "node scripts/generate-build-config.js && npm run build:renderer && electron-builder",
+    "verify:pack-regen": "node scripts/verify-pack-regenerates-build-config.js",
```

`prepack` / `predist` were intentionally left untouched — they handle native binary downloads, which is a different concern. Putting the generator in the script body itself makes the dependency explicit at the script that needs it.
