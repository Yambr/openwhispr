<objective>
Fix CFG-08: `npm run pack` does not regenerate `build-config.generated.{ts,cjs}`, so env-var
overrides set at pack time silently have no effect (the previously-committed values persist).
Same bug exists in `npm run dist`. Root cause: the `prepack` / `predist` lifecycle hooks
chain native compiles + binary downloads but omit the generator step that `prebuild` includes.

Purpose: This is the foundation for every other plan in this phase. PLAN-02..05 verify
their gating by running `npm run pack` with feature-disabling env vars and grepping the
bundle. None of those scenarios are testable until pack regenerates build-config.

Output:
- `package.json` `pack` and `dist` scripts include an explicit `node scripts/generate-build-config.js`
  step immediately before `electron-builder`.
- `scripts/verify-pack-regenerates-build-config.js` — an automated test that runs
  `npm run pack` with a flag override and asserts the generated cjs reflects the override.
</objective>

  <files>scripts/verify-pack-regenerates-build-config.js</files>
  <behavior>
    The test:
    1. Records the current contents of `src/config/build-config.generated.cjs` (or generates a baseline with all defaults).
    2. Manually generates the cjs with all defaults: `node scripts/generate-build-config.js` (no env), then asserts `OAUTH_GOOGLE_ENABLED: true`.
    3. Spawns `npm run pack` with `OPENWHISPR_OAUTH_GOOGLE=false` in env. Wait for completion (no need to wait for full electron-builder if it errors; we only care that the generator ran). Use `--ignore-electron-builder` shortcut: instead of full pack, run only the prepack chain + assert the generator ran. **Simplest implementation:** run `npm run pack -- --help` is not viable; instead use a sentinel approach — touch `src/config/build-config.generated.cjs` to a known timestamp, run `node -e` to invoke `npm run pack` lifecycle hooks only via `npm run prepack` plus the generator invocation that should now be embedded in `pack`. Since prepack is independent and pack's body must include the generator, **the cleanest test is**: spawnSync `bash -c "OPENWHISPR_OAUTH_GOOGLE=false node scripts/run-pack-dry.js"` where `run-pack-dry.js` is a tiny helper that executes the same shell pipeline as `pack` but stops before electron-builder.
    4. **Cleaner approach (chosen):** the test directly runs the literal pack command up to the generator invocation. We assert by:
       - Pre-condition: regenerate build-config with no env → `OAUTH_GOOGLE_ENABLED: true` in cjs.
       - Run: `OPENWHISPR_OAUTH_GOOGLE=false npm run pack` with `electron-builder` stubbed via `ELECTRON_BUILDER_SKIP=1` (we'll have the script no-op when this is set — see Task 2 — OR use a wrapper). To avoid touching electron-builder, the test instead **simulates the pack pipeline** by running each command listed in the `pack` script in order and stopping at electron-builder. It reads `package.json` to get the literal `pack` script string, parses it to find each `&&`-separated step, and runs every step except the final electron-builder one.
    5. After the simulated pack: assert `src/config/build-config.generated.cjs` now has `OAUTH_GOOGLE_ENABLED: false`.
    6. Test 2: Restore default by running generator with no env, assert `OAUTH_GOOGLE_ENABLED: true` (cleanup so dev tree is in good state).
    7. Test fails today because the `pack` script does not contain a generator invocation.

    Test format: pure Node script. Exit code 0 = pass, 1 = fail. Print clear `FAIL: ...` / `PASS: ...` lines for each assertion.
  </behavior>
