<objective>
Flip the OPENWHISPR_STREAMING_ENABLED default from `false` (Phase 04.1 corporate-minimal posture) to `true` (Phase 05 streaming-through-corporate-backend), AND add an auto-disable rule so that a default `npm run build` with no env vars (no backend, no realtime URL) does NOT produce a binary that crashes on first record. Per D-02 in 05-CONTEXT.md, the original Phase 04.1 default was conservative because streaming meant third-party WebSocket connections (AssemblyAI, Deepgram, direct OpenAI). Phase 05 routes streaming through the corporate backend's Speaches+LiteLLM relay, so the privacy-preserving rationale for default-off no longer applies — streaming is now the upstream-default behavior again WHEN a backend is configured.

Purpose: Restore upstream parity for the most common dictation case (live transcription) while keeping `OPENWHISPR_STREAMING=false` as an explicit escape hatch for backends that haven't deployed the realtime relay yet, AND ensuring the default offline build (no backend) does not regress the "default-build-works" principle by silently shipping STREAMING_ENABLED=true with no URL to connect to.

Output: Generator default flipped + auto-disable rule, verify-feature-gating.js scenario matrix updated to assert the new contract (including a default-no-backend scenario that proves auto-disable fires), PROJECT.md CFG-09 amendment documented.
</objective>

The resolveBool() function in generate-build-config.js maps OPENWHISPR_STREAMING (env) to STREAMING_ENABLED (constant). The auto-disable rule in buildResolved() runs AFTER initial resolution so it can see both flags simultaneously.

Auto-disable rule (B1 fix): if `process.env.hasOwnProperty("OPENWHISPR_STREAMING") === false` (caller did not explicitly set it) AND `resolved.OPENWHISPR_STREAMING_ENABLED === true` (default kicked in) AND `resolved.OPENWHISPR_REALTIME_WSS_URL === ""` (no backend, no override), THEN flip `resolved.OPENWHISPR_STREAMING_ENABLED = false`. Explicit `OPENWHISPR_STREAMING=true` with no URL still leaves STREAMING_ENABLED=true (caller's explicit choice; openaiRealtimeStreaming.js's empty-URL guard handles the runtime error).
