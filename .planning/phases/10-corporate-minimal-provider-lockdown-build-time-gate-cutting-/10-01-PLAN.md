<interfaces>
`resolveBool(boolKey)` maps `PROVIDER_LOCKDOWN_ENABLED` → env var
`OPENWHISPR_PROVIDER_LOCKDOWN` automatically (strips `_ENABLED`). Parse rule:
explicit "false" → false; any other set value → true; unset → BOOL_DEFAULTS entry.

`emitTs` / `emitCjs` iterate `BOOL_KEYS` — adding to BOOL_DEFAULTS automatically
emits the constant in both generated modules. No emit-function edit needed.

defaults.ts re-export form (lines 91-106) — MUST use:
`export { PROVIDER_LOCKDOWN_ENABLED } from "./build-config.generated";`
The `export const X = Generated.X` namespace-alias form breaks Rolldown DCE — forbidden.

`prepack` (package.json line 47) currently runs native/download steps but NOT
`node scripts/generate-build-config.js`. `pack` (line 48) runs it explicitly as its
first step. Confirm `pack` regenerates build-config before electron-builder ships
preload — per Phase 04.1-01, the gotcha is that electron-builder copies preload
verbatim, so build-config must be fresh before packaging.
</interfaces>
