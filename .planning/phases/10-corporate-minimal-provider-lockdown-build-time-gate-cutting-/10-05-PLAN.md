<objective>
Under `PROVIDER_LOCKDOWN_ENABLED`, physically remove the BYOK and enterprise
key-management surface below the UI layer: `CustomModelInput`, `ApiKeysService`
(v1/keys), `byokDetection`, the API-key + enterprise-credential IPC channels, and
their preload exposures.

Purpose: PLD-05. Plans 03/04 gated the UI; this plan removes the supporting BYOK
machinery so the corporate bundle carries no key-storage code path and no
enterprise-credential IPC. Mirrors the Phase 04.1 BILLING/REFERRALS preload-submodule
+ `if (BuildConfig.X)` IPC-wrapper pattern.

Output:
- `CustomModelInput.tsx` gated at its consumers (or self-gated) so it DCEs.
- `ApiKeysService.ts` / `byokDetection.ts` consumers gated; under lockdown
  `hasStoredByokKey` resolves to a constant `false`.
- Enterprise key IPC (`get/save-azure-api-key`, `get/save-vertex-api-key`,
  `test-enterprise-connection`, bedrock equivalents) + BYOK key IPC channels
  wrapped in `if (!BuildConfig.PROVIDER_LOCKDOWN_ENABLED)`.
- New `preload-byok.generated.cjs` emitter (analogue of `emitPreloadBilling`)
  gating the BYOK/enterprise key preload methods.
- Settings-store API-key fields kept as typed fields (per CONTEXT discretion —
  keeping the fields keeps store typings honest; the UI/IPC that writes them is
  gone, so they stay permanently empty under lockdown). Document this choice.
</objective>
