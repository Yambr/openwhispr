    # Build-Time Configuration

    ## Overview

    Write a 3–5 paragraph overview that establishes:

    1. What build-time configuration is — variables read at `npm run build` time and baked into the binary; NOT runtime config.
    2. Why build-time, not runtime — smaller attack surface, auditable binaries, no in-app reconfiguration UI (mirrors REQUIREMENTS.md "Out of Scope").
    3. The renderer/main split (per Phase 3 D-01):
       - Renderer reads `import.meta.env.VITE_OPENWHISPR_*` (Vite `define` substitutes literals at build time, enabling DCE).
       - Main process reads `src/config/build-config.generated.cjs` via `require()` (frozen at prebuild time).
       - The user-facing env-var name is always `OPENWHISPR_*` (no `VITE_` prefix). Vite re-exposes a subset to the renderer with `VITE_` prefix internally.
    4. Pointer to source-of-truth files: `src/config/defaults.ts`, `src/config/build-config.generated.cjs`, `electron-builder.config.js`, `scripts/generate-build-config.js`.
    5. Cross-link: "For wire-protocol details of each backend endpoint, see `docs/BACKEND_SPEC.md`. For OAuth provider integration details, see `docs/OAUTH_SPEC.md`."
