<objective>
Fix the local diarization crash "Port at index 0 is not a valid port" (onnxWorker.js:392). Electron `MessagePortMain.postMessage(message, transfer)` accepts ONLY MessagePort objects in `transfer` — unlike the web `MessageChannel`, it rejects ArrayBuffers. The onnx utility worker pushes the embedding ArrayBuffer into the reply transferList, and `speakerEmbeddings.js` pushes the samples ArrayBuffer into the request transferList. Both hops throw. The speaker-embedding step crashes and is swallowed at ipcHandlers.js:8004 ("Speaker embedding extraction skipped"), so cross-meeting speaker labeling produces nothing.

Purpose: Restore local cross-meeting speaker labeling. Diarization STAYS LOCAL (server diarization endpoint was removed in server v1.2.5 — owner-locked decision, do not revisit).

Output: Minimal, upstream-convergent edits to two files (drop the ArrayBuffers from the Electron transferLists so they are structured-cloned instead) plus a regression test that models the real Electron MessagePortMain constraint. Ships as v1.7.22.
</objective>
